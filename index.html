II. How to configure the available projects in your workspace

1. Import the project into Eclipse - top

Step 1: Extract the .zip file that you just downloaded and after that extract the content of the .zip file that is in fact the app (e.g. dmbteam_android_hotel_app_v***.zip). After that create new folder and name it workspace. Start Eclipe and click on File > Switch Workspace > Other and locate the workspace folder that you created and click OK.

Step 2: When you see the Eclipe's Welcome Screen click on File > Import > Android > Existing Android Code into Workspace and click on Next:

Step 3: On the next step you should locate where you extracted the app's .zip file (e.g. dmbteam_android_hotel_app_v***.zip) by clicking the Browse button and then OK:

Step 4: When you click the OK button Eclipse will automatically detect the projects (4) within this folder and will give you possibility to import them. The ObtainApplicationDataActivity item in fact is the Hotel App. On this step, please don't forget to click the checkbox Copy projects into workspace. Then click Finish:

Step 5: Close the Welcome Screen and you will see on the left side the imported projects:

Step 6: Next step is to fix the libraries paths. For that purpose right click on HotelApp project and select Properties > Android and look at the Project Build Target, where you should be sure your target is set to Android 4.2.2. After that look at the Libraries section and remove all items (if you see any) by clicking on it and clicking on Remove button. Do this for all items within the Library section. Then add the libraries (image_loader_library, pinterestlikeview and view_pager_indicator_library) one by one clicking on the Add button and selecting each of the libraries. If you can't see any libraries after you click on the Add button, this means that you have to mark them as libraries. To do so right click on each of them from the Package Explorer on the left and select Properties > Android. Then click on the checkbox Is Library. Then go back to the ObtainApplicationDataActivity (Hotel App), right click > Properties > Android and add the libraries via the Add button. The final variant should be the following:

Step 7: Click on OK and you are ready with the importing. It's good idea to perform clean now. Click on Project > Clean from Eclipse's top navigation and clear all projects.

2. Support Library - top

  • Install the latest version of the the Android Support Library for each project in the workspace. To do so, right click on every project and select Android Tools -> Add Support Lirary.

    1. Customize the information displayed within the application - top

    The application settings can be obtained by either a local file from the assets folder or an online XML file with the same structure as the provided hotel_settings.xml in the assets folder.

    a) Online XML File

    To use a file from an internet URL you should put the URL to the hosted hotel_settings_web.xml file in the HotelXMLTagConstants.java class under the name TAG_URL_SETTINGS

    Just to avoid wrong structure of the xml file, please copy the one located in assets folder and paste it somewhere in different location. After that edit this copied file and upload it to your web server.

    Example:

    public static final String TAG_URL_SETTINGS = "http://dmb-team.com/assets/hotel_settings_web.xml";

    b) Local (Offline) XML File

    In case you want to use the local variant of hotel_settings.xml file to customize the information displayed within the app leave the TAG_URL_SETTINGS empty like so:

    public static final String TAG_URL_SETTINGS = "";

    c) Customize the XML file

    Open hotel_settings.xml file from assets folder (or your copied version, which you want to upload to a web server) and fill data for your hotel. The .xml file is well documented and there are a lot of comments that will help you to understand which information where is displayed within the application.

    d) Online or Internal Reservation Form

    You have the option to use two types of reservation forms:

    1. Your own reservation form (external for the app) located on your website.
    2. Internal reservation form with the following fields: Check-In, Check-Out, Rooms, Adults, Kids, First Name, Last Name, Email and Phone.

    1. Own reservation form (external for the app) located on your website:

    If you want to use your own reservation form you just need to add the url to your reservation form in hotel_settings.xml. Search for element:

    Example:

                    
                http://dmb-team.com

    If you use external reservation form, when the user clicks on the button Book Now within the Reservation Screen, he/she will be redirected to a browser where your external form will be loaded.

    2. Internal reservation form:

    If you want to use internal reservation form provided by our app you have to leave empty the following element within the hotel_settings.xml file:

    Example:

    If the BookOnlinePageUrl element is empty the app will automatically display the internal reservation form. This form includes the following fields: Check-In, Check-Out, Rooms, Adults, Kids, First Name, Last Name, Email and Phone. Please note that when the user have already filled the data in the internal reservation form he/she has to click on a button with name Book Now. After clicking the button the app will load the user's email client and will fill the email's content with the filled data. This way the user can send the needed information to your email address, which you have set in hotel_settings.xml file within the ReservationPageSettings element:

    Example:

                
                n@dhotel.com
